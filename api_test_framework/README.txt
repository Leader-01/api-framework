当前所有文件（配置文件，公共方法，测试用例，数据，报告，log）都在项目根目录下，随着用例的增加和功能的补充，
文件会越来越多，不便于维护和管理，因此我们要建立不同的文件夹，对文件进行分类组织

1.在项目中新建以下文件夹：
    1.1 config: 存放项目配置文件
    1.2 data: 存放用例数据文件
    1.3 lib: 公共方法库
    1.4 log: 存放日志文件
    1.5 report: 存放报告文件
    1.6 test: 存放测试用例
         1.6.1 user: 存放user模块用例 （模块下要有__init__.py，这样里面的用例才能读取到）
